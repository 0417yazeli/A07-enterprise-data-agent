# A07 比赛项目排期与分工（4 周 / 3 人）

> **比赛**：企业数据底座智能问析 Agent 系统
> **演示形式**：现场答辩 + 现场演示 + 录屏视频
> **团队规模**：3 人 · 学生 / 初学者
> **截止时间**：T+28 天
> **更新时间**：每次周会后刷新

---

## 1. 核心策略

| 维度 | 策略 |
|------|------|
| **范围** | 优先闭环 4 大核心能力，业务知识管理 + 数据资源理解 + NL2SQL + 可视化 + ML 至少各做扎实 1 个演示场景 |
| **演示** | **现场能跑 > 完美代码**。必须准备"录制兜底 + LLM 离线兜底 + 数据快照兜底"三层备份 |
| **学习** | 每周前置 0.5-1 天学习缓冲（LangChain / Vue3 / PostgreSQL 各 1-2 天） |
| **协作** | 每人独立主攻一个智能体领域 + 跨域互助；每天 20:00 同步 10 分钟 |
| **风险** | 关键路径单一负责人 + 平行备份人；外部依赖（LLM API）双供应商 |

## 4. 演示兜底方案（**关键**）

现场演示最大杀手是 **网络 / LLM API 不稳定 / 现场电脑环境差异**。准备三层兜底：

### 第一层：正常路径
现场电脑 + 现场网络 + 真实 LLM API 演示

### 第二层：LLM 降级
- **方案 A**：预生成 20 条自然语言问句 + 预生成 SQL + 预计算结果（缓存到 JSON）
- **方案 B**：本地小型模型（如 Qwen 1.5B int4 量化）作为兜底
- 触发条件：API 连续 2 次失败 / 响应 > 10s

### 第三层：录屏兜底
- 5 分钟精简版录屏，演示核心流程
- 触发条件：现场电脑故障 / 网络完全断开

### 现场环境要求
- [ ] 现场电脑预装：Python 3.11+、Node 20+、PostgreSQL 14+
- [ ] 现场预热：开 2 个浏览器标签（前端 + LangSmith 可选）
- [ ] 现场预加载：4 个核心查询的预计算结果

## 6. 跨域技术对接契约

### 前后端接口契约（Day 1 必须定稿）
- **健康检查**：`GET /api/v1/health` → `{status, version, db_ok, llm_ok}`
- **业务知识**：`GET/POST/PUT/DELETE /api/v1/knowledge/topics`
- **数据资源**：`GET /api/v1/schema/tables`、`GET /api/v1/schema/tables/{name}`
- **自然语言问析**：`POST /api/v1/chat/message` → 流式 SSE
- **ML 训练**：`POST /api/v1/ml/train`、`POST /api/v1/ml/predict`

### 数据库表清单（A 在 D3 输出）
- 生产域 4 张：workorder / process / route / shift
- 质量域 3 张：inspection / defect / standard
- 设备域 4 张：equipment / downtime / alarm / maintenance
- 库存域 4 张：material / balance / transaction / warehouse
- 公共维度 3 张：product / line / date

### 前端页面清单（B 在 D1 输出）
1. 主页（Dashboard）：核心问数入口 + 快速分析卡片
2. 业务知识管理：列表 + 详情 + CRUD
3. 数据资源理解：表树 + 字段说明 + 样例
4. 自然语言问析：聊天 UI + 结果展示
5. ML 建模：训练 + 预测 + 解释
6. 知识图谱：关系图可视化
